Tips

undefined和null类型：

undefined：

当变量声明未初始化时，typeof检测变量值为undefined；当变量未声明时不能进行除typeof之外的其他操作，且对未声明的变量检测typeof时值也是undefined；–>所以，建议声明变量时同时初始化，让其区别于变量未声明却typeof时的undefined。

null:

null逻辑上表示空对象指针，为typeof传入null、空数组字面量、空对象字面量时返回object；在声明将来要保存对象值的变量时建议使用null进行初始化，这样只需要检查变量类型是不是null就能知道变量有没有被重新赋予一个对象的引用。

总结：

null是空对象指针，undefined只是变量未初始化。

浮点数：

科学计数法：

整数值E/e次幂==>0.3e4–>3000

精确数位：

计算机中的浮点数精确到小数点后17位，所以算术运算小数点后的值并没有整数精确。例如：0.1 + 0.2 !==0.3；因此不建议测试某个特定浮点值。

ECMAscript数值范围：

JavaScript能表示的数值范围在 [min_value，max_value] 中，超过最小值时转为-infinity；超过最大值时转为Infiniti；表示负/正无穷大，他们都不能继续参与运算。

检测数值是否在可表示范围内时，可以使用isFinite(数值)，返回值是布尔值；在可表示范围内返回true，否则返回false。

JS执行机制

扩展知识：

0、JavaScript是单线程程序处理机制。HTML5提出允许JavaScript创建多个子线程，但其完全被主线程操控且不能操作DOM，所以其本质还是单线程。

1、进程：一个具有独立功能的程序在一个数据集上的一次动态执行的过程，进程是能拥有资源和独立运行的最小单位，也是程序执行的最小单位。

  • 动态性：进程是程序的一次执行过程，是临时的，有生命期的，是动态产生，动态消亡的；
  • 并发性：任何进程都可以同其他进程一起并发执行；
  • 独立性：进程是系统进行资源分配和调度的一个独立单位；
  • 结构性：进程由程序、数据和进程控制块三部分组成。

2、线程：线程是程序执行中一个单一的顺序控制流程。

3、区别：

  • 一个进程可以有一个或多个线程，各个线程之间共享程序的内存空间(即所在进程的内存空间)。一个标准的线程由线程ID、当前指令指针(PC)、寄存器和堆栈组成。而进程由内存空间(代码、数据、进程空间、打开的文件)和一个或多个线程组成。
  • 线程是程序执行的最小单位，而进程是操作系统分配资源的最小单位；
  • 一个进程由一个或多个线程组成，线程是一个进程中代码的不同执行路线；
  • 进程之间相互独立，但同一进程下的各个线程之间共享程序的内存空间(包括代码段、数据集、堆等)及一些进程级的资源(如打开文件和信号)，进程与进程之间互不可见；
  • 调度和切换：线程上下文切换比进程上下文切换要快得多。

4、浏览器的一些主要进程：

  • 浏览器进程：控制界面显示，用户界面交互等；
  • 第三方插件进程：管理插件使用；
  • GPU进程：3D绘制硬件加速；
  • 渲染进程：页面解析、代码执行、页面渲染；
  • 渲染进程下的线程：
    • GUI渲染线程：解析html，css构建DOM树，布局等；与JS解析引擎线程互斥
    • JS引擎：负责处理JS脚本，浏览器同时只能有一个JS引擎线程在运行JS。因为与GUI互斥，JS可能会阻塞页面渲染。
    • 定时器线程：setInterval，setTimeout
    • 事件触发线程：当事件触发时将事件推到处理队列等待主线程执行。
    • 异步请求线程：当XMLHttpRequest 链接后浏览器开启线程，检测请求状态变更，将回调函数推入任务队列。

执行机制：

0、执行代码，发现异步交给宿主环境。JS引擎执行同步代码，宿主环境执行异步代码。

1、同步任务与异步任务进入不同的执行场所：同步–主线程，异步– event table。

2、异步任务执行完毕将回调函数推入任务队列；

3、等待主线程执行完毕，读取任务队列中的函数进入主线程执行。

4、event loop：主线程执行完毕（执行栈为空）并查找任务队列的过程重复执行。

5、JS引擎的monitoring process进程会不断检测执行栈是否为空。

宏任务/微任务

异步任务又被划分为宏任务和微任务，不论宏任务微任务，任务队列中存放的都是他们的回调函数，等待主线程查找。

宏任务：由宿主环境发起的任务，包括全局代码，setInterval,setTimeout…

微任务：由js自身发起，包括promise.then(),process.nextTick()…

执行：

0、执行同步代码，执行完毕后查找任务队列，进入微任务队列查看有没有可以执行的微任务，推入主线程执行；

1、微任务执行完毕后去宏任务队列查找可执行的宏任务，推入主线程执行；

2、主线程为空进入任务队列继续查找，循环执行代码。

禁用效果

1、表单元素禁用：disabled = true

2、非表单元素：pointer-events = none

非表单元素设置css属性pointer-events，让所有鼠标操作失效，该属性默认为auto。

使用场景：按钮样式不好修改，开发中一般使用a链接代替按钮。

获取表单数据

当表单项特别多时，可以使用第三方库或新版本JS内置对象FormData一次性获取所有表单数据并存储为对象返回。

一、form-serialize:

1、下载serialize的JS库文件并引入；

2、语法：serialize(form表单对象，{hash:布尔值})

当布尔值为true，将表单数据收集组织为对象；对象中的数据都是字符串格式。

当布尔值为false，将表单数据收集组织为get请求后拼接的参数形式的字符串。

属于第三方库，不能用于上传文件。

二、FormData:

新版本JS内置对象FormData，也是一个构造函数。

1、实例化对象：

new FormData() //html有Form表单元素但没有Form标签。

new FormData(form标签对象) // html有form标签

2、常用方法：参考MDN

.get(name) 返回给定name属性的值

.entries() 返回包含所有键值对的对象,也看不到具体键值对

注意：不能一开始就获取，要表单项通过验证以后在触发提交事件时才获取，这时候的参数值才是需要提交给接口的参数。一开始就获取的话值为空，接口报错422，参数错误。

表单–文件上传：

1、accept 属性，过滤选择的文件后缀，当点击选择文件时只出现accept中过滤的后缀文件

2、change事件，当表单上传的文件发生变化时触发change事件

3、e.target.files 获取上传的文件对象，返回伪数组，通过下标获取具体文件。

注意：当第一次选择文件之后，第二次选择文件时点击取消，此时files从有值变成了空，也会触发change事件。一般情况下这次触发change没有意义，所以先判断e.target.files[0]中是否有值才进行后续操作。

扩展：

文件上传按钮设置CSS样式时，文字图片也随之改变，即无法单独设置样式。所以开发中一般将其隐藏，使用别的按钮代替触发file的默认点击事件。

token

在登录时，如果不走登陆页面而直接地址栏输入地址会发生不需要验证用户名密码也能进入主页的情况。逻辑上这样是不允许的。

解决办法：token认证；

1、用户第一次登录时服务端返回token并存储，用户的每次页面跳转和请求操作都需要在请求头中携带token，由服务端验证token真伪时效。

2、需要token认证的界面：前端判断本地是否有token，后端路由守卫验证是否携带了token。（前端自查，后端阻止）前端自查意义不大，可以携带假token。
